# QERP v2

QERP v2는 **배포 가능한 모의 투자 애플리케이션**을 목표로 합니다. 로컬 데모가 아니라, 실제 운영 환경에 점진적으로 올릴 수 있는 구조를 처음부터 유지합니다.

## 제품 비전
- 현대 리테일 브로커리지 앱 스타일의 최소 UX 제공
- 실시간 시세 기반의 종이거래(페이퍼 트레이딩) 경험 제공
- 비전문 사용자도 활용 가능한 Quant 보조 경험 제공
- 기능 과잉보다 **명확한 범위와 배포 가능성** 우선

## 타겟 사용자
- 실전 자금 투입 전 투자 흐름을 연습하려는 개인 사용자
- 퀀트 개념은 익숙하지 않지만 신호/설명을 원하는 초중급 사용자
- 간단한 투자 앱 아키텍처를 빠르게 검증하려는 개발 팀

## 코어 기능 (MVP 기준)
- 실시간 시장 데이터 조회 및 기본 차트
- 매수/매도 페이퍼 주문 생성 및 체결 처리
- 포트폴리오/손익 추적
- 일반 모드 ↔ 퀀트 모드 전환
- 비전문가용 퀀트 보조 인사이트(설명 중심)

## MVP 비목표 (Non-goals)
- 실계좌 연동/실주문 라우팅
- 고급 백테스트 엔진
- 소셜 트레이딩/커뮤니티
- 옵션/선물/파생상품 거래
- 멀티 리전 고가용성 및 초고성능 최적화

## 상위 아키텍처
- `frontend/`: 사용자 UI, 차트, 주문/포트폴리오 화면, 모드 토글
- `backend/`: **Java + Spring Boot** 기반 API, 주문/포트폴리오 도메인 로직, 데이터 저장
- `quant-worker/`: **Python** 기반 시세 지표 계산, 신호 생성, 설명 텍스트 생성
- `infra/`: 배포/환경 구성 문서 및 IaC 기반 자산 관리 위치
- `docs/`: 범위, 결정사항, 로드맵 등 프로젝트 기준 문서

자세한 구조는 `docs/architecture.md`를 참고합니다.

## 단계별 로드맵
1. **Phase 0 — 프로젝트 정렬**
   - 레포 구조 정리, MVP 범위/결정 문서화
2. **Phase 1 — 세로 슬라이스 MVP**
   - 시세 표시 → 주문 생성/체결 → 포트폴리오 반영까지 단일 흐름 구현
3. **Phase 2 — 퀀트 모드 추가**
   - Quant Worker 연동, 단순 신호 + 자연어 설명 제공
4. **Phase 3 — 배포 안정화**
   - 스테이징 배포, 모니터링/로그, 운영 체크리스트 보강
5. **Phase 4 — 확장 준비**
   - 기능 확장 전 성능 병목 측정 및 우선순위 재정의
